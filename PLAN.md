# План развития rust_snippets

## Цель
Создать разнообразную библиотеку Rust-сниппетов, покрывающую основные темы языка и экосистемы.

## Задачи (по темам)

- [x] Асинхронность (tokio, async/await, futures)
- [ ] Работа с файлами (read/write, символьные/бинарные)
- [ ] JSON (serde, сериализация/десериализация)
- [ ] Сети (TCP/UDP, HTTP-клиент/сервер)
- [ ] Многопоточность (threads, channels, sync primitives)
- [ ] CLI (clap, аргументы, подкоманды)
- [ ] Конфиги (toml/yaml/json, validation)
- [ ] Логгирование (tracing, log, env_logger)
- [ ] Тестирование (unit, integration, fixtures)
- [ ] Обработка ошибок (thiserror, anyhow, custom errors)
- [ ] Паттерны (builder, observer, pipeline, async-stream)
- [ ] Утилиты (conversion, macros, derive)

## Процесс
Каждый heartbeat OpenClaw автоматически:
1. Синхронизирует репозиторий с GitHub.
2. Запускает агента для создания нового сниппета по невыполненной теме.
3. Коммитит и пушит изменения.

## Приоритеты
1. База: асинхронность, файлы, JSON, сети.
2. Продвинутые паттерны и утилиты.
3. Примеры с реальными сценариями (CLI-инструменты, серверы, парсеры).
